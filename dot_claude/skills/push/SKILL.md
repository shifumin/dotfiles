# Push

変更内容をcommitしてpushする。

## 処理フロー

### 1. 状態確認

`git status` と `git diff` で変更を確認。変更がない場合は「変更なし」と報告して終了。

### 2. コミット粒度の判断

CLAUDE.md「Git > コミットメッセージ」のConventional Commits形式に従う。

| 状況 | 対応 |
|------|------|
| 単一種類の変更 | 1コミットにまとめる |
| 複数種類が混在 | 種類ごとに選択的にステージし、別コミットにする |

例: `docs` の変更と `feat` の変更がある場合 → `docs` のファイルを先にステージしてコミット → 次に `feat` のファイルをステージしてコミット

### 3. コミット作成

各コミットで以下を実行:

1. 対象ファイルを `git add` でステージ
2. `git diff --staged` で秘匿情報がないか確認
3. 秘匿情報を検出した場合 → ユーザーに報告し、コミットを中止
4. コミットメッセージを作成してコミット

### 4. Push

| 結果 | 対応 |
|------|------|
| 成功 | 完了報告 |
| 失敗 | エラー内容を報告し、対処法を提案 |
