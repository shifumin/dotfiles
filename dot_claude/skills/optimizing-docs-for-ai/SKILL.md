# Optimizing Docs for AI

ドキュメントをAIが正確に解釈・実行できるように最適化する。

引数 `$ARGUMENTS` でドキュメントパスを受け取る。省略時はAskUserQuestionで確認。

## 5つの観点

| 観点 | 説明 | チェック項目 |
|------|------|--------------|
| **簡潔性** | 不要・冗長・重複を削る | 自明な内容、同じ説明の複数記載 |
| **明示性** | 条件・理由・例外を明確に | 理由のない指示、暗黙の条件分岐 |
| **構造化** | 見出し・箇条書き・テーブルで整理 | `#`→`##`→`###`順、`-`統一、比較テーブル化、30行超セクション |
| **十分性** | 目的達成に必要な情報が揃っているか | 不足している前提・手順・例外 |
| **一貫性** | 用語・表現を統一 | 同じ概念への異なる名称 |

## 処理フロー

1. Readツールでドキュメント全体を読み込む
2. 5つの観点で改善点を特定
3. 追加・削除が必要、またはセクションの目的が不明確な場合 → AskUserQuestionで確認
4. Editツールで改善を適用
5. 変更内容を観点別に箇条書きで報告

## 制約事項

| 種別 | 内容 |
|------|------|
| 禁止 | 元の意図を変更する |
| 禁止 | 推測で情報を追加 |
| 禁止 | 既存の命名規則を無視した構造変更 |
| 必須 | 元のドキュメントを読み込んでから編集 |
| 必須 | 改善の根拠を示せるようにする |
