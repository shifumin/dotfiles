# optimize-for-ai [ドキュメントパス]

指定されたドキュメントをAI向けに最適化する。

---

## 基本方針

| 方針 | 説明 |
|------|------|
| シンプルさ優先 | 冗長な表現を削る |
| 自明な内容は省略 | 文脈から推測可能な説明は不要 |
| 元の意図を保持 | 目的を損なわない |
| 曖昧さを排除 | 条件・理由・例外を明確に |
| 具体例は最小限 | 曖昧な場合のみ追加 |

---

## 実行手順

### Step 1: 分析

1. Readツールでドキュメント全体を読み込む
2. 目的・構成を把握し、改善点を特定

### Step 2: 改善チェック

#### 構造・フォーマット

- 見出し階層が `#` → `##` → `###` の順か
- 箇条書き記号が統一されているか（`-` 推奨）
- 長いセクション（30行超）は分割検討
- 比較情報はテーブル化

#### 文脈・意図の明確化

- 理由のない指示には「なぜ」を1文追加
- 条件分岐を明示
- 優先度を示す（🔴必須 / 🟡推奨）

### Step 3: 内容レビュー

ドキュメントの目的に対して内容が適切か評価する。

| 観点 | 確認事項 |
|------|----------|
| 必要性 | 目的達成に不要な項目はないか |
| 十分性 | 目的達成に不足している項目はないか |
| 重複 | 同じ内容が複数箇所にないか |
| 一貫性 | 用語・表現が統一されているか |

**ユーザー確認が必要な場合**:
- 追加・削除すべき項目がある
- 目的が不明確

→ AskUserQuestionで確認してから進める

### Step 4: 編集

Editツールで改善を適用し、変更内容を記録

---

## 出力フォーマット

```

**対象ファイル**: [パス]

### 改善内容

#### 構造・フォーマット
- [項目]

#### 文脈・意図の明確化
- [項目]

#### 内容レビュー
- [項目]
```

---

## 制約事項

### 禁止

- 元の意図を変更する
- 不必要な絵文字や深い見出し階層を追加
- 推測で情報を追加
- 既存の命名規則を無視した構造変更

### 必須

- 元のドキュメントを読み込んでから編集
- 改善の根拠を示せるようにする
