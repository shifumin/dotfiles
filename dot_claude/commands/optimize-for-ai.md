# optimize-for-ai [ドキュメントパス]

ドキュメントをAI向けに最適化する。

## 基本方針

| 方針 | 説明 |
|------|------|
| シンプルさ優先 | 冗長な表現を削る |
| 自明な内容は省略 | 文脈から推測可能な説明は不要 |
| 元の意図を保持 | 目的を損なわない |
| 曖昧さを排除 | 条件・理由・例外を明確に |

## 実行手順

### 分析

1. Readツールでドキュメント全体を読み込む
2. 目的・構成を把握し、改善点を特定

### 改善チェック

**構造・フォーマット**:
- 見出し階層が `#` → `##` → `###` の順か
- 箇条書き記号が統一されているか（`-` 推奨）
- 長いセクション（30行超）は分割検討
- 比較情報はテーブル化

**文脈・意図の明確化**:
- 理由のない指示には「なぜ」を1文追加
- 条件分岐を明示

### 内容レビュー

| 観点 | 確認事項 |
|------|----------|
| 必要性 | 目的達成に不要な項目はないか |
| 十分性 | 目的達成に不足している項目はないか |
| 重複 | 同じ内容が複数箇所にないか |
| 一貫性 | 用語・表現が統一されているか |

追加・削除が必要、または目的が不明確な場合 → AskUserQuestionで確認

### 編集

Editツールで改善を適用し、変更内容をカテゴリ別に報告

## 制約事項

| 種別 | 内容 |
|------|------|
| 🔴禁止 | 元の意図を変更する |
| 🔴禁止 | 推測で情報を追加 |
| 🔴禁止 | 既存の命名規則を無視した構造変更 |
| 🟡必須 | 元のドキュメントを読み込んでから編集 |
| 🟡必須 | 改善の根拠を示せるようにする |
