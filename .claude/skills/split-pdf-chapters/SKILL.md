# PDFチャプター分割

PDFファイルのアウトライン（目次/ブックマーク）構造に基づいて、チャプター単位で個別PDFファイルに分割する。

## 引数

| 引数 | 必須 | 説明 |
|------|------|------|
| PDFパス | Yes | 分割するPDFファイルのパス |
| depth | No | 分割する階層レベル（デフォルト: 1。1=章, 2=節, 3=項） |
| オプション | No | `--force`, `--verbose`, `--complete`（後述） |

引数不足時はAskUserQuestionで確認する。

---

## 処理フロー

### 1. PDF存在確認

- ファイルの存在と`.pdf`拡張子を確認
- 存在しない場合はエラーメッセージを表示して終了

### 2. 分割実行

**実行コマンド:**

```bash
mise exec -C ~/ghq/github.com/shifumin/pdf-chapter-splitter -- \
  bundle exec ruby ~/ghq/github.com/shifumin/pdf-chapter-splitter/pdf_chapter_splitter.rb \
  [OPTIONS] "<PDFパス>"
```

**CLIオプション:**

| Option | 説明 | 使用場面 |
|--------|------|----------|
| `-d N` | 分割する階層レベル | ユーザーが「セクション単位」「節ごと」と言った場合は2以上を指定 |
| `-f` | 既存の chapters/ ディレクトリを上書き | 再分割時に使用 |
| `-v` | 詳細出力 | デバッグや確認が必要な時 |
| `-c` | 次のセクションまでの全ページを含める | depth 2以上で漏れなく分割したい時 |
| `-n` | dry-run（プレビューのみ） | ユーザーが「確認だけ」「プレビュー」と言った時 |

### 3. 出力先

PDFファイルと同じディレクトリに `chapters/` ディレクトリが作成される。

**出力ファイル命名:**
- `000_前付け.pdf` — 最初のチャプターより前のページ
- `001_Chapter_Title.pdf` — 各チャプター（3桁ゼロ埋め連番）
- `999_付録.pdf` — 最後のチャプター以降のページ

### 4. 完了報告

分割後、作成されたファイル一覧（`ls chapters/`）を表示して報告する。

---

## エラー処理

| エラーパターン | 対処 |
|---------------|------|
| PDFが存在しない / 拡張子が`.pdf`でない | エラーメッセージを表示して終了 |
| PDFにアウトライン（目次）がない | ユーザーに報告して終了 |
| `chapters/` ディレクトリが既に存在する | `--force` オプションの使用を案内するか、確認の上 `-f` 付きで再実行 |
